Security

AI Security

Threats to test

  • Prompt injection and tool-use manipulation.
  • Training data leakage, sensitive output disclosure, and memorization.
  • Adversarial inputs, jailbreaks, and unsafe content pathways.
  • Dependency, vendor, plugin, and retrieval pipeline risk.

Control evidence

Security audit work should include abuse-case design, test logs, mitigation records, monitoring signals, incident handling, and boundaries for human review.
